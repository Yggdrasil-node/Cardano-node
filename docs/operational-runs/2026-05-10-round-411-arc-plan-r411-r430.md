## D1 ‚Äî Cardano.Timeseries vendor or equivalent

**Recommendation: Option C (defer entirely).**

Local availability: vendored at
`.reference-haskell-cardano-node/bench/cardano-timeseries-io/src/Cardano/Timeseries/`
‚Äî **1,528 lines across 7 leaf files + 4 subdirs**. Component.hs
alone is 137 lines and depends on `Cardano.Logging` (unported),
`Store s Double` typeclass (Elab.hs, 894 lines), and a
query-language Interp (365 lines).

Why C:
- Upstream's `for (hasTimeseries config) (const $ Timeseries.create ...)`
  is `Maybe`-shaped ‚Äî Run.hs proceeds when `hasTimeseries=None`.
- Operators run cardano-tracer fully functionally without timeseries
  (Prometheus + Grafana covers the same Q&A surface).
- Vendoring 1,528 lines + embedded query language is multi-quarter
  scope; outside the cardano-tracer arc.
- R426 ships TimeseriesServer.hs as a routing-only shell that
  returns `404 Not Found` for `/timeseries/*` when handle is
  `None` (always-None per Option C).

## D2 ‚Äî Full TLS termination

**Recommendation: Option A (`axum-server` 0.7).**

Defer to R429 (post-Run.hs supervisor). Justification:
- `axum-server` is the canonical TLS adjacent crate to axum 0.8;
  same maintainer.
- Transitive surface fully overlaps lettre's `tokio1-rustls` feature
  ‚Äî `rustls`, `webpki-roots`, `rustls-pki-types` already in lock.
- R408 `load_pem_certs` / `load_pem_key` produce DER-encoded byte
  vectors directly consumable by
  `axum_server::tls_rustls::RustlsConfig::from_der(certs, key)`.
- **Not Run.hs-blocking**: operators with `epForceSSL=Just True`
  get `TracerMissingCertificate` traces today; R429 wires real TLS.

## D3 ‚Äî EKG-equivalent metrics surface

**Recommendation: Option A (inline `MetricsStore`).**

Critical path because `runEKGAcceptor` consumes
`(EKG.Store, TVar MetricsLocalStore)` per node ‚Äî Run.hs cannot ship
otherwise. The inline port covers:
- Per-node `Arc<RwLock<BTreeMap<MetricName, MetricValue>>>` store.
- `MetricValue` enum (Counter/Gauge/Label) mirroring
  `System.Metrics.ReqResp.MetricValue`.
- Counter/Gauge/Label registration + setters.
- `render_prometheus` exposition renderer (lift from
  `node/src/tracer.rs::MetricsSnapshot::to_prometheus_text` shape).
- `render_ekg_html` monitoring page renderer.
- `metrics_help.json` parser feeding `te_metrics_help`.

Schema-flexible (`BTreeMap` by name) ‚Äî cardano-tracer is *passive*,
storing whatever names the forwarding mini-protocol delivers from
upstream nodes. No name-locked schema needed.

Vendoring `prometheus-client` (Option B) adds ~12 transitive deps
for behavior we already have at lower cost. Option C breaks Run.hs
end-to-end behavior with a real upstream node.
