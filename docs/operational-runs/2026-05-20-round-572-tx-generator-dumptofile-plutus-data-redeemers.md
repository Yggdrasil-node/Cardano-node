# Round 572 tx-generator DumpToFile TxDats + Redeemers

Date: 2026-05-20

## Scope

This round lifts the `plutus_data` and `redeemers` boundary in
`show_alonzo_witness_set`. Previously, any Plutus-bearing transaction
generated by `gen_tx` (which R555 enabled for script spends) would fail
`SubmitMode::DumpToFile` with a generic "non-vkey witnesses" error.
After this round, those transactions render into upstream-structured
`MkTxDats` and `MkRedeemers` Show output, including blake2b_256
hashes computed from the upstream-canonical CBOR shape.

Native scripts, bootstrap witnesses, and Plutus V1/V2/V3 script-witness
bytes remain on explicit `TxGenError` boundaries until their downstream
mirrors land.

## Changes

- Replaced the old "non-vkey witnesses" rejection in
  `show_alonzo_witness_set` with proper rendering of `plutus_data` and
  `redeemers`. The witness set is now accepted whenever native scripts,
  bootstrap witnesses, and Plutus V1/V2/V3 script-witness bytes are
  empty.
- Added `show_plutus_data` matching upstream stock-derived
  `Show (PV1.Data)`:
  - `Constr <i> [...]` — alternative integer + showsPrec-11-wrapped fields
  - `Map [...]` — list of `(key,value)` tuples
  - `List [...]` — list of items
  - `I <n>` — integer with parens around negatives at p=11
  - `B <bytestring>` — Latin1 byte-string Show
- Added `show_haskell_bytestring` rendering bytes as upstream Haskell
  `Show ByteString`: printable ASCII inline (excluding `"` and `\`,
  which are backslash-escaped), `\n`/`\t`/`\r` named escapes, bytes
  >= 0x80 as `\NNN` decimal escapes, with `\&` separator inserted
  before any following digit so the escape boundary is unambiguous.
  Note: the 0x00-0x1F control-character mnemonic escapes (`\NUL`,
  `\SOH`, ..., `\DEL`) are not yet rendered; structural Show parity
  holds but full byte parity needs a future round.
- Added `show_alonzo_tx_dats` rendering
  `MkTxDats (TxDatsRaw {unTxDatsRaw = fromList [...]} (blake2b_256:
  SafeHash "<hex>"))`. Map entries are sorted by DataHash byte-lex
  (matching upstream `Data.Map (DataHash, Data era)` ordering) and the
  outer hash is computed over `tag 258 + definite-length array of
  each datum's canonical CBOR`, matching upstream
  `encodeWithSetTag . Map.elems . unTxDatsRaw`. Empty input still
  produces the previous fixture (`[0xd9, 0x01, 0x02, 0x80]` hash).
- Added `show_alonzo_redeemers` rendering
  `MkRedeemers (RedeemersRaw {unRedeemersRaw = fromList [...]}
  (blake2b_256: SafeHash "<hex>"))`. Map entries are sorted by `(tag,
  index)` (matching upstream `Map PlutusPurpose AsIx era`) and the
  outer hash is computed over a definite-length array of
  `[tag, index, data, ex_units]` redeemer entries, matching
  `Redeemer::encode_cbor`. Empty input produces the previous fixture
  (`[0x80]` hash).
- Added `show_alonzo_plutus_purpose` (tag 0/1/2/3 →
  `AlonzoSpending` / `Minting` / `Certifying` / `Rewarding`) and
  `show_alonzo_ex_units` (`ExUnits {exUnitsMem = ..., exUnitsSteps =
  ...}` record).
- Added 7 focused unit tests covering integer signs, byte-string
  escape paths (printable / non-ASCII / escape-boundary / backslash /
  double-quote), List, Map+Constr nesting, single Spending redeemer,
  multi-redeemer `(tag, index)` sort, and single-datum TxDats
  envelope.
