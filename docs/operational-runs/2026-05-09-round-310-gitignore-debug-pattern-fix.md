# Round 310 — `.gitignore` over-broad `debug` pattern fix (CI failure)

**Date:** 2026-05-09  
**Branch:** `main`  
**Predecessor:** [`R309`](2026-05-09-round-309-agents-md-arc-closure.md)  
**Trigger:** CI failure on `cargo fmt --all -- --check` post-R309 push.

## Symptom

CI run reported:

```text
Run cargo fmt --all -- --check
Error writing files: failed to resolve mod `check_node_configuration`:
/home/runner/work/Cardano-node/Cardano-node/crates/cardano-cli/src/era_independent/debug/check_node_configuration.rs
does not exist
Error: Process completed with exit code 1.
```

The local working tree built and tested clean throughout R294–R309
because the file existed locally. The 12-file `era_independent/debug/`
subtree was created during R294 but never made it into git history.

## Root cause

Root `.gitignore` line 2 was a bare `debug` pattern intended to ignore
Cargo's `target/debug/` build output. Without a leading slash or
`**/` prefix, gitignore rules match files or directories with that
basename **anywhere** in the tree:

```text
$ git check-ignore -v crates/cardano-cli/src/era_independent/debug/check_node_configuration.rs
.gitignore:2:debug    crates/cardano-cli/src/era_independent/debug/check_node_configuration.rs
```

`crates/cardano-cli/src/era_independent/debug/` is the strict 1:1
mirror of upstream `Cardano/CLI/EraIndependent/Debug/` (R294 closure).
The over-broad pattern silently swallowed the entire subtree:

| Tracked at HEAD before R310 | Created locally R294 |
|---|---|
| `crates/cardano-cli/src/era_independent/debug.rs` (parent shell) | `debug/check_node_configuration.rs` + `debug/check_node_configuration/{command,run}.rs` |
| | `debug/log_epoch_state.rs` + `debug/log_epoch_state/{command,run}.rs` |
| | `debug/transaction_view.rs` + `debug/transaction_view/{command,run}.rs` |
| | `debug/{command,option,run}.rs` |

Total: 12 .rs files (~6.7 KB combined) silently ignored.

The parent shell `era_independent/debug.rs` was tracked at HEAD with
`pub mod check_node_configuration; pub mod log_epoch_state;
pub mod transaction_view; pub mod command; pub mod option; pub mod run;`,
so on a fresh CI checkout (or any clone) the `cargo fmt` invocation
attempts to walk into the missing children and fails.

The local strict-mirror gate (`scripts/check-strict-mirror.py`) +
`docs/strict-mirror-audit.tsv` were already aware of all 12 files
(they were graded at R294's discovery + hand-grade pass) — the gate
walks the local working tree, not git history, so the divergence
between "what the local tree has" and "what the index has" was
invisible to it.

A similar over-broad bare pattern existed for `target` (line 3); both
were redundant given line 5's `**/target/`. Lines 2 and 3 also
duplicated each other since `target/debug/` is wholly contained in
`target/`. R310 collapses the four entries (`debug` / `target` /
`tmp` / `**/target/`) to three (`/target/` / `**/target/` / `tmp/`).

## Fix

Two-step:

**Step 1 — `.gitignore` cleanup.** Replace the over-broad bare
patterns with anchored / glob-prefixed equivalents:

```diff
 # Generated by Cargo
-debug
-target
-tmp
+/target/
 **/target/
+tmp/
```

The `/target/` prefix anchors to repo root (Cargo's actual location);
`**/target/` keeps the existing safety-net for nested workspaces; the
trailing `/` on `tmp/` scopes the rule to directories named `tmp`.
After the fix, `git check-ignore -v` against any of the 12 files
returns "not ignored".

**Step 2 — re-add the 12 R294 files.** `git add
crates/cardano-cli/src/era_independent/debug/` stages the previously-
invisible subtree. `cargo fmt --all` confirms the files were already
in canonical rustfmt form (R294 had run fmt locally before commit;
the format was good — only the staging step had been silently
skipped). All five gates remain at the 4,855-test baseline.

## Hardening — pattern audit

Surveyed every other bare-basename ignore pattern that could swallow
source-tree dirs:

| Pattern | Source-tree match risk | Mitigation status |
|---|---|---|
| `debug` (line 2 pre-R310) | `crates/cardano-cli/src/era_independent/debug/` | **Fixed in R310** — replaced with anchored `/target/`. |
| `target` (line 3 pre-R310) | None today; would match any `target/` source dir. | **Removed in R310** as redundant; covered by `**/target/`. |
| `tmp` (line 4 pre-R310 → `tmp/` post-R310) | None today. | Scoped to dirs named `tmp` only. |
| `**/db/`, `**/chain/`, `**/ledger/` (lines 41–43) | `**/ledger/` does match `crates/ledger/`; explicit `!crates/ledger/` exception on line 45 already protects. | Verified — surveyed `find` for source-tree `db/` / `chain/` directories; none exist. |
| `wallet/`, `metadata/`, `keys/`, `certs/` (lines 47–50) | None today. | Verified — surveyed `find`; no source-tree directories with those names. |

R310 only removes the broken `debug` and redundant `target` lines;
the remaining over-broad rules either have explicit exceptions or
have no current basename collision with source-tree directories.

## Lessons

- **Bare gitignore patterns are dangerous in repositories that mirror
  upstream directory structure.** Cardano's upstream tree has
  `Cardano/CLI/EraIndependent/Debug/`; Yggdrasil's strict-mirror
  policy maps that to `crates/cardano-cli/src/era_independent/debug/`.
  Cargo's gitignore convention assumes `target/debug/` is the only
  `debug` of interest. The two collide at the bare-pattern boundary.
- **The strict-mirror gate is local-tree-aware, not index-aware.**
  `scripts/check-strict-mirror.py` walks the working tree filesystem,
  not `git ls-files`. A future enhancement could cross-check against
  `git ls-files` to catch divergences between the local tree and the
  index (the silent-ignore failure mode of R310). Deferred — R310's
  pattern fix removes the immediate failure mode, and the index-vs-tree
  cross-check is broader scope (would also catch missing-file scenarios
  unrelated to ignore patterns).
- **CI is the canonical truth on what's actually shipped.** The local
  working tree built clean across 35 R-arc rounds (R274–R309) while
  HEAD was missing 12 .rs files. CI surfaced the divergence at the
  first push of a doc-only round (R309) — exactly the kind of "trivial
  PR exposes hidden problem" event that justifies pushing every round
  to CI rather than batching multiple rounds.
